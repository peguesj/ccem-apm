## US-006 - Implement AG-UI event streaming endpoint
- Created ApmV4.EventStream GenServer with sequence counter, event storage (500-item cap), and PubSub broadcasting on topic "ag_ui:events"
- Created ApmV4Web.AgUiController with SSE endpoint at GET /api/ag-ui/events using chunked transfer encoding
- SSE response sets text/event-stream Content-Type, cache-control: no-cache, x-accel-buffering: no headers
- Initial STATE_SNAPSHOT emitted on connect containing current agents, sessions, and notifications
- Event loop receives PubSub messages and forwards as SSE `id:/event:/data:` formatted lines
- Keepalive comment sent every 30 seconds to prevent connection timeout
- agent_id query parameter filters events to specific agent (global events like STATE_SNAPSHOT always pass through)
- Integrated with AgentRegistry: register_agent emits RUN_STARTED, update_status to "completed"/"finished" emits RUN_FINISHED
- All 9 AG-UI event types supported: TEXT_MESSAGE_START/CONTENT/END, TOOL_CALL_START/ARGS/END, STATE_SNAPSHOT, RUN_STARTED, RUN_FINISHED
- Convenience emitter functions generate unique IDs: run_id (run-*), message_id (msg-*), tool_call_id (tc-*) using crypto.strong_rand_bytes
- EventStream added to Application supervision tree between AgentRegistry and Endpoint
- Route added: GET /api/ag-ui/events in /api scope
- Files changed: lib/apm_v4/event_stream.ex (new), lib/apm_v4_web/controllers/ag_ui_controller.ex (new), lib/apm_v4/agent_registry.ex (modified), lib/apm_v4/application.ex (modified), lib/apm_v4_web/router.ex (modified), test/apm_v4/event_stream_test.exs (new), test/apm_v4_web/controllers/ag_ui_controller_test.exs (new)
- 28 new test cases, 80/80 total pass, zero compiler warnings
- Learnings: Phoenix SSE is best implemented with send_chunked/2 + chunk/2 and a receive loop, not with Phoenix Channels. The AG-UI protocol maps naturally to SSE with id/event/data fields. Process.whereis/1 guards in AgentRegistry prevent crashes when EventStream hasn't started yet (e.g., during early boot or isolated tests). Testing SSE controllers requires Task.async + Task.shutdown pattern since the connection blocks indefinitely.
