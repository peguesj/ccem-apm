## US-003 - Port HTTP API endpoints to Phoenix controllers
- Created ApmV4Web.ApiController with 5 actions: status, register, heartbeat, agents, notify
- GET /api/status returns JSON with uptime, agent_count, session_id, server_version
- POST /api/register accepts agent_id (or id), name, tier, status, deps, metadata; returns 201 on success, 400 on missing ID
- POST /api/heartbeat accepts agent_id + status, updates last_seen; returns 404 for unknown agent, 400 for missing ID
- GET /api/agents returns JSON array of all registered agents
- POST /api/notify accepts title, message, level; creates notification with defaults for missing fields
- Added :api pipeline routes in router.ex under /api scope
- Stored server_start_time in Application env for uptime calculation
- Added clear_all/0 to AgentRegistry for safe test isolation (avoids GenServer.stop which disrupted supervision tree)
- Updated AgentRegistryTest to use clear_all/0 instead of stop/restart pattern
