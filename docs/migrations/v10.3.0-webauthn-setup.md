# v10.3.0 — WebAuthn FIDO2 Setup (auth-v10.3-s1 / CP-298)

This migration enables hardware-attested approval decisions on
`POST /api/v2/approvals/:id/approve` using FIDO2/WebAuthn. Built on
[`wax_`](https://hex.pm/packages/wax_) per DRTW `01-authorization.md` GAP 8.

## Why

Before v10.3.0, the approval endpoint trusted any caller in possession of an
API token. WebAuthn binds an approval decision to a **physical authenticator**
(touchID / Windows Hello / YubiKey) so impersonation requires possession of
the device — not just credential exfiltration.

## Configuration

The gate is opt-in. Set the runtime flag once approvers have provisioned
authenticators:

```elixir
# config/runtime.exs
config :apm_v5, :require_webauthn_for_approval, true
```

Default (`false`) preserves the v9.x behavior for back-compat.

## Registration flow (browser)

```js
// 1. Fetch a server-issued challenge
const { challenge, rp, user, pubKeyCredParams } =
  await fetch("/api/v2/auth/webauthn/registration_challenge", { method: "POST" })
    .then(r => r.json());

// 2. Ask the authenticator to create a credential
const cred = await navigator.credentials.create({
  publicKey: {
    challenge: Uint8Array.from(atob(challenge), c => c.charCodeAt(0)),
    rp,
    user: {
      id: Uint8Array.from(user.id, c => c.charCodeAt(0)),
      name: user.name,
      displayName: user.displayName
    },
    pubKeyCredParams,           // [{ type: "public-key", alg: -8 }, { type: "public-key", alg: -7 }]
    authenticatorSelection: { userVerification: "required" },
    timeout: 60000
  }
});

// 3. Send the attestation to the server
await fetch("/api/v2/auth/webauthn/register", {
  method: "POST",
  headers: { "content-type": "application/json" },
  body: JSON.stringify({
    user_id: user.name,
    attestation_object: b64url(new Uint8Array(cred.response.attestationObject)),
    client_data_json:   b64url(new Uint8Array(cred.response.clientDataJSON))
  })
});
```

## Assertion flow (approval-time)

```js
// 1. Ask the authenticator to sign a challenge
const assertion = await navigator.credentials.get({
  publicKey: {
    challenge: Uint8Array.from(atob(challenge), c => c.charCodeAt(0)),
    allowCredentials: [{ type: "public-key", id: credId }],
    userVerification: "required",
    timeout: 60000
  }
});

// 2. Submit the assertion alongside the approval
await fetch(`/api/v2/approvals/${gateId}/approve`, {
  method: "POST",
  headers: { "content-type": "application/json" },
  body: JSON.stringify({
    user_id: "alice",
    approver: { user_id: "alice" },
    webauthn_assertion: {
      credential_id:       b64url(new Uint8Array(assertion.rawId)),
      signature:           b64url(new Uint8Array(assertion.response.signature)),
      authenticator_data:  b64url(new Uint8Array(assertion.response.authenticatorData)),
      client_data_json:    b64url(new Uint8Array(assertion.response.clientDataJSON))
    }
  })
});
```

When `:require_webauthn_for_approval` is `true`, approvals without a valid
assertion respond `401 { error: "webauthn attestation required", reason: ... }`.

## Replay protection

Each `authenticatorData` blob carries a 32-bit big-endian `sign_count`. The
server tracks the last accepted count per credential and rejects any value
≤ stored. This is the FIDO-mandated cloned-authenticator detector
(W3C WebAuthn §6.1.1).

## Supported algorithms

| COSE alg | Curve     | Use                          |
| -------- | --------- | ---------------------------- |
| -8       | Ed25519   | YubiKey 5 FIPS, recent macOS |
| -7       | P-256     | TouchID, Windows Hello       |

## Audit events

Both registration and successful assertion emit AuditLog entries:

- `:webauthn_registered` — actor=user_id, resource=`credential:<hex>`
- `:webauthn_verified` — includes the new `sign_count`

## Failure modes

| HTTP | reason                  | Meaning                                            |
| ---- | ----------------------- | -------------------------------------------------- |
| 401  | `missing_assertion`     | Policy enabled, but body has no `webauthn_assertion` |
| 401  | `missing_user_id`       | No `user_id` provided in body                       |
| 401  | `no_credential`         | User has zero registered authenticators            |
| 401  | `credential_not_found`  | `credential_id` not in store (phishing / wrong key) |
| 401  | `replay_detected`       | `sign_count` did not strictly increase             |
| 401  | `bad_signature`         | Ed25519 / ECDSA verify failed                      |
| 401  | `malformed_auth_data`   | Authenticator data shorter than 37 bytes           |
| 401  | `malformed_base64`      | One of the assertion fields wasn't valid base64url |
