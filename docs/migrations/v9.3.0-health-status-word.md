# v9.3.0 Health Endpoint Migration: "ok" → "pass"

The `/api/health` endpoint status word changed from `"ok"` to `"pass"` in v9.3.0 to comply with IETF draft-inadarei-api-health-check-06 (Content-Type `application/health+json`).

## Affected endpoint
- `GET /api/health` returns `{"status": "pass"|"warn"|"fail", ...}` (was `{"status": "ok"}`)
- `GET /.well-known/health` (new RFC 8615 alias) — same payload

## Unchanged for back-compat
- `GET /api/status` still returns `{"status": "ok", ...}` for legacy consumers

## Client action required
Clients reading `response.status == "ok"` from `/api/health` MUST update to `response.status == "pass"`. Alternative: read `response.status in ["pass", "warn"]` for healthy-or-degraded check.

Reference: hc-s1 (CP-250) shipped in v9.3.0 Governance Foundation sprint.
